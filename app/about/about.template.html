Overview

This weather app a programming example built on AngularJS 1.5.11. It uses the OpenWeatherApp API 0.1.6. It borrows the basic design of the Bing weather app on Windows desktop. I have tested it only on Chrome.

The app is functional and the code is fairly tight, but this was a quick demonstration project. Some of the first things I would do if this were a real-world project are: convert "wind" and "barometer" values into something more meaningful; various refactorings; test on multiple browswers.

Main Features

  • One button calls both the current-weather API and the 5-day forecast API
  • The user can input either a city name or a zip code. The app detects whether a zip code has been entered and forms its call to the api accordingly
  • The user can also pick a city from a dropdown of previously saved cities
  • Cities can be saved to the user's local storage
  • The same city is not saved multiple times
  • The "Save City" button is disabled when there is no city to save

Technical Points of Interest

  • Integrates third-party weather icons, mapping the OpenWeather codes to the appropriate icons
  • Modifies the incoming weather data for ease-of-use in the markup
  • Implements routing
  • Uses Angular Resource rather than $http
  • Implements AngularJS components instead of controllers
  • Is installable via 
    npm install
  • Is runnable using 
    npm start
  • Uses dynamically loaded webfonts
